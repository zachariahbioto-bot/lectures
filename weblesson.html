The Web: Browsers, Servers, and HTTP

The web operates on the client-server model. A client, typically a web browser like Chrome or Firefox, makes a request, and a server responds with the requested information. This communication is governed by the Hyper Text Transfer Protocol (HTTP).

 Web Browsers (The Client)

A web browser is a software application that allows you to access and view websites. It's the client in the client-server relationship. When you type a URL (Uniform Resource Locator) into your browser's address bar, you're initiating a request. The browser's job is to:

  • 

    Request a webpage from a server.

  • 

    Receive the data from the server, which is typically in the form of HTML, CSS, and JavaScript files.

  • 

    Interpret and render this data to display the webpage on your screen.

HTML provides the structure, CSS adds the styling, and JavaScript handles the interactivity.

HTML Structure

Based on the book "HTML and CSS: Design and Build Websites" by Jon Duckett, HTML describes the structure of a web page using markup. Just as a newspaper or a form has a structured layout, HTML provides a way to structure content on a webpage.

 Key Concepts

  • 

    HTML Pages are Documents

    HTML pages are essentially text documents. HTML provides a way to structure content on a webpage.

  • 

    Tags and Elements

    HTML uses tags, or elements, to give content special meaning. For example, the <p> tag is for a paragraph, and <h1> is for a main heading. Tags usually come in pairs, an opening tag and a closing tag.

  • 

    Hierarchical Layout

    The use of headings and subheadings reflects a hierarchy of information, similar to how a word processor organizes content.

  • 

    The <head> and <body>

    The entire visible content of a webpage is contained within the <body> element. Before the <body>, there is a <head> element that holds information about the page, like its title, which appears in the browser tab.

HTML Lists

HTML provides three types of lists to help you organize information on a web page, each with a specific purpose.

Unordered Lists (`<ul>`)

These are used for lists where the order of items is not important. By default, browsers display list items with a bullet point. 

<ul>
  <li>Item 1</li>
  <li>Item 2</li>
</ul>

Ordered Lists (`<ol>`)

These are for lists where the sequence of items matters, such as a set of instructions or a countdown. The browser will automatically number each list item. 

<ol>
  <li>First step</li>
  <li>Second step</li>
</ol>

Definition Lists (`<dl>`)

This type of list is used to present a list of terms and their corresponding definitions. It consists of definition term (`<dt>`) and definition description (`<dd>`) elements. 

<dl>
  <dt>Term</dt>
  <dd>Definition of the term</dd>
</dl>

Example:

Here is a simple code example showing an unordered list for groceries and an ordered list for a recipe. 

<!-- Unordered List Example -->
<h3>Shopping List</h3>
<ul>
  <li>Milk</li>
  <li>Bread</li>
  <li>Eggs</li>
</ul>

<!-- Ordered List Example -->
<h3>How to Make Toast</h3>
<ol>
  <li>Put bread in the toaster.</li>
  <li>Press the lever.</li>
  <li>Wait for it to pop.</li>
</ol>

Chapter 4: Links

This chapter would likely cover how to create links between web pages using HTML. Links, or hyperlinks, are the fundamental building blocks of the web, allowing users to navigate from one page to another.

Key Concepts

  • The `<a>` Tag

    The primary element for creating a link is the `<a>` (anchor) tag. The content within this tag becomes the clickable text or image.

    <a href="https://www.google.com">Google</a>

  • The `href` Attribute

    This is the most important attribute of the `<a>` tag. The `href` (Hypertext Reference) attribute specifies the destination URL or file path that the link will point to.

  • Absolute vs. Relative URLs

    • Absolute URLs: These are full web addresses, including the protocol and domain name, such as `https://www.example.com/page.html`. You use them to link to external websites.
    • Relative URLs: These are used to link to other pages within the same website. They specify the path from the current page to the target page, such as `about.html` or `../images/photo.jpg`.

  • Linking to Sections of a Page

    You can also use links to jump to specific sections within the same web page. This is done by assigning an `id` to the target element and then linking to that `id` using a `#` symbol in the `href` attribute. For example: `<a href="#section-2">Go to Section 2</a>`.

  • Email and Phone Links

    Links can also be used to open a user's email client or initiate a phone call. For an email link, the `href` is set to `mailto:`, and for a phone link, it's `tel:`.

Example:

Here is an example of a simple link to a website and a link that opens an email. 

<!-- Link to an external website -->
<a href="https://www.example.com">Visit Example.com</a>

<!-- Email link -->
<a href="mailto:test@example.com?subject=Hello%20there!">Send us an email</a>

Chapter 5: Images

This chapter would likely explain how to embed images into a web page using HTML. Images are crucial for making web pages visually appealing and informative.

 Key Concepts

  • 

    The `<img>` Tag

    The primary element for inserting an image is the `<img>` tag. Unlike the `<a>` and other tags, this is a self-closing tag, meaning it doesn't have a separate closing tag.

    <img src="path/to/image.jpg" alt="Description of image">
  • 

    The `src` Attribute

    This attribute is essential and stands for "source." It specifies the path to the image file you want to display. This can be either a relative or absolute URL.

  • 

    The `alt` Attribute

    This is a very important attribute for accessibility. The `alt` (alternative text) attribute provides a text description of the image. This text is displayed if the image fails to load or if a user is using a screen reader. It's crucial for users with visual impairments and for search engines that can't "see" the image.

  • 

    Height and Width

    The `height` and `width` attributes can be used to set the dimensions of the image. While it's generally better to control image dimensions using CSS, these attributes can be used directly in the HTML tag to reserve the space for the image, preventing the page from shifting as it loads.

    <img src="image.jpg" alt="A beautiful landscape" width="500" height="300">
  • 

    Image Formats

    • JPEG: Best for photographs and images with lots of colors.
    • PNG: Best for images that require transparency or have sharp lines, like logos and icons.
    • GIF: Best for simple animations or images with a limited color palette.

Example:

Here is an example of an image with `alt` text and specified dimensions. 

<!-- Image Example -->
<img src="https://placehold.co/400x200/5C7AEA/FFFFFF?text=Web+Concepts" alt="A placeholder image for a web concepts lesson" width="400">

Chapter 6: Tables

This chapter would likely explain how to create and structure tables in HTML to display tabular data. While tables were once used for page layout, their modern purpose is strictly for presenting data in a structured way, like a spreadsheet.

 Key Concepts

  • 

    The `<table>` Tag

    This is the main container for the entire table.

    <table>...</table>
  • 

    Table Rows (`<tr>`)

    Each row in the table is created using the `<tr>` (table row) tag.

    <tr>...</tr>
  • 

    Table Data (`<td>`)

    This tag creates a cell within a table row and contains the actual data.

    <td>...</td>
  • 

    Table Headers (`<th>`)

    These are used for the heading cells of a table, often displayed in bold and centered by default. They are used to describe the data in a column or row.

    <th>...</th>
  • 

    Table Structure (`<thead>`, `<tbody>`, `<tfoot>`)

    These tags are used to organize the table's content into a header, body, and footer. This is important for accessibility and for styling purposes.

  • 

    `colspan` and `rowspan`

    These attributes are used to create cells that span across multiple columns (`colspan`) or multiple rows (`rowspan`). This is useful for creating more complex table layouts.

Example:

Here is an example of a simple table with a header row and two data rows. 

<!-- Simple Table Example -->
<table class="w-full text-left">
  <thead>
    <tr>
      <th class="p-2 border-b-2">Name</th>
      <th class="p-2 border-b-2">Age</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td class="p-2">Alice</td>
      <td class="p-2">25</td>
    </tr>
    <tr>
      <td class="p-2">Bob</td>
      <td class="p-2">30</td>
    </tr>
  </tbody>
</table>

Chapter 7: Forms

This chapter would likely explain how to create **forms** in HTML to collect user input. Forms are a critical part of many websites, enabling everything from logging in and searching to submitting feedback and making purchases.

 Key Concepts

  • 

    The `<form>` Tag

    This is the main container for all form elements. The `<form>` tag has important attributes like `action`, which specifies where to send the data, and `method`, which determines how to send it (usually `GET` or `POST`).

  • 

    Input Elements (`<input>`)

    The `<input>` tag is the most common form element. It's a self-closing tag with a `type` attribute that defines what kind of input it is. Common types include `text` for a single-line text field, `password` for a text field where the characters are hidden, `email` for an email address field, `radio` for a single selection from a group of options, `checkbox` for multiple selections, and `submit` for a button that sends the form data.

  • 

    The `<label>` Tag

    The `<label>` tag is used to associate a text label with a form input. This is crucial for accessibility, as clicking the label will focus the input field, and screen readers can read the label aloud.

  • 

    Other Form Elements

    • `<textarea>`: Used for multi-line text input, such as comments or messages.
    • `<select>` and `<option>`: Used to create a dropdown menu with a list of options.
    • `<button>`: A versatile element used to create a clickable button.

Example:

Here is a simple login form with a username and password field. 

<!-- Simple Login Form -->
<form action="/login" method="post">
  <div>
    <label for="username">Username:</label>
    <input type="text" id="username" name="username" required>
  </div>
  <div>
    <label for="password">Password:</label>
    <input type="password" id="password" name="password" required>
  </div>
  <button type="submit">Log In</button>
</form>

Chapter 8: Extra Markup

This chapter covers a variety of less common but still useful HTML tags that provide more specific meaning (or **semantics**) to your content. These tags help give additional context to text that goes beyond a simple paragraph or heading, which can be beneficial for accessibility, search engine optimization (SEO), and overall code clarity.

 Key Concepts

  • 

    Blockquote (`<blockquote>`) and Quote (`<q>`) tags

    These tags are used for quotations. **`<blockquote>`** is for longer quotes that are displayed as a separate block of text, while **`<q>`** is for shorter, inline quotes. Using these tags properly distinguishes quoted material from your own writing. You can also use the `<cite>` tag within a blockquote to provide the source of the quote.

    <blockquote>
  <p>The future belongs to those who believe in the beauty of their dreams.</p>
  <cite>- Eleanor Roosevelt</cite>
</blockquote>

<p>As my teacher often said, <q>Never give up on your passion.</q></p>
  • 

    Abbreviations and Acronyms (`<abbr>`) tag

    The `<abbr>` tag is used to specify an abbreviation or acronym. You can provide the full text of the abbreviation in a `title` attribute, which appears as a tooltip when a user hovers over the abbreviated text.

    <p>I'm learning <abbr title="HyperText Markup Language">HTML</abbr> for web development.</p>
  • 

    Code (`<code>`) tag

    This tag is used to display a short piece of computer code. Browsers typically render the text in a monospaced font to make it distinct from the rest of the text.

    <p>The <code> </code> tag is a good choice for showing inline code.</p>
  • 

    Superscript (`<sup>`) and Subscript (`<sub>`) tags

    These tags are used to display text as superscript or subscript, respectively. They are useful for things like mathematical formulas (e.g., x�) or chemical formulas (e.g., HO).

    <p>The formula for water is H<sub>2</sub>O.</p>
<p>A common equation is E = mc<sup>2</sup>.</p>
  • 

    Definitions (`<dfn>`) tag

    The `<dfn>` tag is used to indicate the defining instance of a term. This helps clarify the meaning of a word or phrase within a paragraph.

    <p>A <dfn>web server</dfn> is a computer that stores web content and responds to requests from web browsers.</p>

Example:

Here is a combined example showing how you can use multiple extra markup tags in a single paragraph to provide more context. 

<!-- Combined Example -->
<p>
  The formula for sulfuric acid is H<sub>2</sub>SO<sub>4</sub>, which is very different from water (H<sub>2</sub>O). 
  A common phrase attributed to Albert Einstein is, <q>Imagination is more important than knowledge.</q>
</p>

Chapter 9: Flash, Video & Audio

This chapter explains how to embed multimedia content like video and audio into a web page. While Flash is now obsolete, the principles for embedding video and audio are fundamental to modern web design.

 Key Concepts

  • 

    The `<video>` and `<audio>` Tags

    These are the modern way to embed video and audio files directly into a web page without relying on third-party plugins. They are supported by all major browsers.

    • `<video>`: Used to embed a video player. Important attributes include `src`, `controls`, `width`, and `height`.
    • `<audio>`: Used to embed an audio player. Similar to video, it uses the `src` and `controls` attributes.
  • 

    The `<source>` Element

    Since different browsers support different video and audio formats, the `<source>` element is used inside the `<video>` or `<audio>` tags to specify multiple file formats. The browser will then play the first format it can support.

  • 

    Accessibility

    It's important to provide alternative content for users who cannot access the multimedia. You can place text within the `<video>` or `<audio>` tags that will be displayed if the browser does not support the element.

  • 

    Controlling Playback with JavaScript

    You can create a custom UI and control the media programmatically using JavaScript. Common methods include `play()`, `pause()`, and properties like `currentTime` and `volume`.

Example:

Here is a simple example showing how to embed a video and an audio file. 

<!-- Video Example -->
<video width="320" height="240" controls>
  <source src="movie.mp4" type="video/mp4">
  Your browser does not support the video tag.
</video>

<!-- Audio Example -->
<audio controls>
  <source src="sound.mp3" type="audio/mpeg">
  Your browser does not support the audio tag.
</audio>

Chapter 10: HTML5 Structure

This chapter introduces the new structural elements that were added in HTML5. The goal of these new tags is to provide more **semantic meaning** to the layout of a web page, replacing generic `

` tags. This makes the code easier to read for both developers and search engines.

 Key Concepts

  • 

    Header (`<header>`)

    Used for the introductory content of a page or a section, often containing the site logo, a main heading, or navigation links.

  • 

    Navigation (`<nav>`)

    Specifically for a block of navigation links. It's a key tool for accessibility, as screen readers can use it to easily find the main navigation.

  • 

    Section (`<section>`)

    A generic sectioning element for grouping related content, such as a chapter, a set of articles, or a part of a page. It should always have a heading to describe its content.

  • 

    Article (`<article>`)

    Represents a self-contained piece of content that could be distributed independently, like a blog post, a news story, or a forum post. It should make sense on its own.

  • 

    Aside (`<aside>`)

    For content that is tangentially related to the main content, such as a sidebar, a block of related links, or a callout box. It can be separated from the main content without losing meaning.

  • 

    Footer (`<footer>`)

    Used for the footer of a document or a section. It typically contains information like copyright notices, contact details, or links to related documents.

Example:

Here is how these semantic tags work together to create a simple, well-structured page layout. 

<body>
  <!-- Page Header -->
  <header>
    <h1>My Awesome Website</h1>
    <nav>
      <a href="/">Home</a>
      <a href="/blog">Blog</a>
      <a href="/contact">Contact</a>
    </nav>
  </header>

  <!-- Main Content -->
  <main>
    <section>
      <h2>Latest Articles</h2>
      <article>
        <h3>First Blog Post</h3>
        <p>This is a self-contained blog post about...</p>
      </article>
      <article>
        <h3>Second Blog Post</h3>
        <p>This is another post about a different topic.</p>
      </article>
    </section>
    
    <!-- Sidebar -->
    <aside>
      <h3>Related Links</h3>
      <ul>
        <li><a href="/about">About Us</a></li>
      </ul>
    </aside>
  </main>

  <!-- Page Footer -->
  <footer>
    <p>© 2023 My Awesome Website. All rights reserved.</p>
  </footer>
</body>
